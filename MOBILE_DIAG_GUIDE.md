# 📱 移动端诊断使用指南

## 🎯 快速访问

### 在手机浏览器访问：
```
https://echo-platform-eta.vercel.app/mobile-diag.html
```

---

## 📋 诊断步骤

### 1️⃣ 自动检测
页面会自动检测并显示：
- ✅ 设备信息（手机型号、系统）
- ✅ 网络状态（WiFi/4G/5G）
- ✅ 资源加载情况
- ✅ 浏览器功能支持

### 2️⃣ 功能测试

**测试主应用：**
```
点击 "测试主应用" 按钮
→ 检查主页是否能正常访问
→ 查看返回的状态码
```

**测试网络：**
```
点击 "测试网络请求" 按钮
→ 检查是否能访问外部 API
→ 排除网络限制问题
```

### 3️⃣ 清除缓存

如果遇到问题：
```
点击 "清除缓存" 按钮
→ 清除所有缓存数据
→ 刷新页面重试
```

## 📸 如何发送诊断信息

### 方法 1：一键生成报告（推荐）

1. **点击 "报告问题" 按钮**
2. **诊断报告会自动复制到剪贴板**
3. **粘贴发送给开发者**

报告包含：
```
🔍 ECHO 诊断报告
- 时间
- URL
- 设备信息
- 网络状态
- 诊断日志
```

### 方法 2：截图

**需要截图的内容：**

1. **诊断页面首页**
   - 设备信息卡片
   - 网络状态卡片
   - 资源加载卡片

2. **诊断日志**
   - 向下滑动到"诊断日志"卡片
   - 截图所有日志内容

3. **测试结果**
   - 点击各个测试按钮后的提示

### 方法 3：录屏

**录制操作步骤：**
```
1. 打开诊断页面
2. 滑动查看所有卡片
3. 点击各个测试按钮
4. 录制整个过程
```

## 🔍 常见问题自测

### 问题 1：诊断页面都打不开

**可能原因：**
- 网络问题
- Vercel 部署问题
- 域名被墙

**解决方法：**
```
1. 检查其他网站能否访问
2. 切换 WiFi/4G 网络
3. 等待 5 分钟后重试
```

---

### 问题 2：诊断页面正常，主页打不开

**可能原因：**
- React 渲染问题
- JavaScript 执行错误
- 资源加载失败

**查看诊断日志：**
```
向下滑动到"诊断日志"部分
查看是否有红色错误
截图发送给开发者
```

---

### 问题 3：测试主应用失败

**可能原因：**
- 路由配置问题
- 服务器配置问题
- 文件路径问题

**解决方法：**
```
1. 查看返回的状态码
2. 截图错误信息
3. 发送给开发者
```

## 📊 诊断信息说明

### 设备信息
```
用户代理：浏览器和系统信息
平台：操作系统
语言：系统语言
屏幕：屏幕分辨率
视口：实际显示区域

触摸：是否支持触摸
内存：设备内存
核心数：CPU 核心数
网络：在线状态
连接：网络类型

✅ 网络连接正常 - 可以访问网络
📶 网络类型 - 4G/5G/WiFi

```

### 资源加载
```
✅ CSS 样式 - 样式表加载正常
✅ 本地存储 - localStorage 可用
✅ 会话存储 - sessionStorage 可用
✅ Canvas - 画布功能正常
✅ Fetch API - 网络请求功能正常
```

## 💡 使用技巧

### 技巧 1：使用无痕模式
```
在 Safari 中：
1. 长按链接
2. 选择"在无痕浏览中打开"

可以排除缓存和 Cookie 干扰
```

### 技巧 2：强制刷新
```
iOS Safari:
1. 长按刷新按钮
2. 选择"清空缓存并重新载入"

Android Chrome:
1. 长按刷新按钮
2. 选择"清空缓存并重新载入"
```

### 技巧 3：保存诊断页面
```
1. 添加到主屏幕
2. 下次直接从桌面访问
3. 方便随时诊断
```
