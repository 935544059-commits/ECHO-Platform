# ECHO 智能体接入配置中心 - 测试流程指南

## 📋 完整测试流程

### 步骤 1: 填写智能体基本信息
1. **上传头像**：点击蓝色头像区域上传图片（支持 jpg/jpeg/png，最大 2M）
2. **输入名称**：在"智能体名称"输入框填写名称，例如：`客服助手`
3. **添加标签**：输入标签后按 Enter 键，例如：`客服`、`AI`、`自动回复`
4. **能力描述**：填写描述（最多 100 字），例如：`用于处理用户咨询的智能客服助手`

### 步骤 2: 配置 API 基本信息
1. **API 轮询频率**：选择 `每 1h`（默认）
2. **API 类型**：选择 `知 + 工作流` 或 `九天工作流`
3. **API 名称**：填写 API 名称，例如：`Dify 客服 API`
4. **API 地址**：填写第三方 MAAS 平台 API 地址
   - 示例（Dify）：`https://api.dify.ai/v1/chat-messages`
   - 示例（Coze）：`https://api.coze.com/v1/chat`
5. **API 描述**：可选填写
6. **认证类型**：选择 `API_KEY` 或 `BEARER_TOKEN`
7. **API_KEY/Token**：填写你的密钥

### 步骤 3: 配置入参动态构建
1. **请求体模板**：编辑 JSON 模板
   ```json
   {
     "query": "{{USER_INPUT}}",
     "session_id": "{{SESSION_ID}}",
     "user_id": "{{USER_ID}}"
   }
   ```
2. **点击变量拾取器**中的变量，自动插入到模板中
3. **添加业务参数**（可选）：
   - 点击"添加业务参数"
   - 填写参数键名（如：`scene`）和显示标签（如：`业务场景`）
   - 选择参数类型（文本/数字/下拉选择/开关）

### 步骤 4: 配置出参解析与映射
1. **先进行接口测试**：在 API 配置区填写测试输入，点击"接口测试"
2. **查看响应预览**：右侧会显示 API 返回的 JSON 数据
3. **点击 JSON 树节点**：选择需要的字段
4. **添加输出变量**：
   - 系统自动生成变量名（如：`var_text`）
   - 选择数据类型（文本/图片链接/图表数据/JSON 对象）
   - 点击"添加输出变量"

### 步骤 5: 保存配置
1. 点击底部 **直接保存** 按钮
2. 或点击 **接口测试** 再次验证

## 🎯 快速测试（无真实 API）

如果你想快速体验界面交互，可以：

### 方案 1: 使用 Mock API
1. 访问 https://jsonplaceholder.typicode.com/
2. 使用示例 API：`https://jsonplaceholder.typicode.com/posts/1`
3. 方法选择 `GET`
4. 点击"接口测试"查看响应

### 方案 2: 使用本地 Mock 服务
创建一个简单的 Node.js 服务：

```javascript
// mock-server.js
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({
    data: {
      output: {
        text: '这是一个测试回复',
        images: ['https://example.com/image.jpg']
      }
    }
  }));
});

server.listen(3000, () => {
  console.log('Mock server running at http://localhost:3000');
});
```

### 方案 3: 使用在线 API 测试工具
- https://httpbin.org/ - 测试 HTTP 请求
- 使用端点：`https://httpbin.org/post`
- 方法：`POST`
- 可以看到完整的请求响应
