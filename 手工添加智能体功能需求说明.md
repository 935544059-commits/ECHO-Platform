# 手工添加智能体功能需求说明

## 一、页面结构

### 1. 整体布局
- **顶部导航栏**：包含返回按钮和页面标题「手工添加智能体」
- **主内容区**：包含多个配置卡片，垂直排列
- **底部操作栏**：包含保存按钮，固定在页面底部

### 2. 核心组件

| 组件名称 | 位置 | 功能描述 |
|---------|------|----------|
| AgentInfoPanel | 主内容区顶部 | 智能体基本信息配置，包括名称、标签、描述、头像 |
| ApiBasicConfig | 主内容区中部 | API 基本信息配置，包括 API 类型、URL、认证方式等 |
| InputParamsConfig | 主内容区中部 | 入参动态构建，包括参数映射、业务参数配置、JSON 预览 |
| OutputMappingConfig | 主内容区底部 | 出参解析与映射，包括渲染插槽配置、响应预览 |

### 3. 辅助组件

| 组件名称 | 功能描述 |
|---------|----------|
| ValidationErrors | 错误提示组件，显示校验失败的错误信息 |
| SuccessToast | 成功提示组件，显示操作成功的信息 |
| BusinessParamModal | 业务参数配置弹窗，用于添加/编辑业务参数 |
| DynamicFormPreview | 动态表单预览，展示业务参数在对话端的 UI 效果 |
| RequestPreview | 请求预览，展示最终发送的请求体结构 |
| MarkdownRenderer | Markdown 渲染器，支持图片和图表的智能渲染 |

## 二、交互逻辑

### 1. 保存操作

**触发条件**：点击底部操作栏的「保存配置」按钮

**校验规则**：
- 智能体名称不能为空
- API 名称不能为空
- API 地址不能为空
- 如果选择 API_KEY 认证，API_KEY 不能为空
- 如果选择 BEARER_TOKEN 认证，Bearer Token 不能为空
- 必须配置一个关联了 {{USER_INPUT}} 的入参
- 必须配置了「主回复」的出参

**反馈逻辑**：
- 校验失败：显示红色错误提示框，列出所有错误信息，5秒后自动消失
- 接口未测试：显示确认提示框，询问是否继续保存
- 保存成功：显示绿色成功提示框，3秒后自动消失

### 2. 业务参数配置

**触发条件**：点击「添加业务参数」按钮

**交互流程**：
1. 弹出业务参数配置弹窗
2. 填写参数信息（变量标识、显示文本、交互类型等）
3. 点击「保存」按钮，参数添加到列表中
4. 实时更新 JSON 预览和对话端 UI 预览

### 3. 出参映射配置

**触发条件**：点击插槽的「拾取」按钮

**交互流程**：
1. 在左侧 JSON 树中选择对应节点
2. 自动绑定路径到插槽
3. 实时更新渲染预览

### 4. 实时预览

**触发条件**：配置发生变化时

**反馈逻辑**：
- 入参配置：实时更新 JSON 预览
- 业务参数：实时更新对话端 UI 预览
- 出参配置：实时更新渲染预览

## 三、数据结构

### 1. 智能体基本信息

```javascript
agentInfo: {
  name: '',          // 智能体名称
  tags: [],          // 标签数组
  description: '',   // 描述
  avatar: null,      // 头像
}
```

### 2. API 配置

```javascript
apiConfig: {
  queryFrequency: '1h',               // 查询频率
  apiType: 'dify',                    // API 类型
  apiName: '',                        // API 名称
  apiUrl: '',                        // API 地址
  apiDescription: '',                 // API 描述
  authType: 'API_KEY',                // 认证类型
  apiKey: '',                        // API Key
  bearerToken: '',                    // Bearer Token
  method: 'POST',                     // 请求方法
  headers: [{ key: 'Content-Type', value: 'application/json' }], // 请求头
  body: '{
  "query": "{{USER_INPUT}}"
}', // 请求体
  isStreaming: false,                 // 是否流式响应
}
```

### 3. 入参配置

```javascript
inputParams: {
  bodyTemplate: '{
  "query": "{{USER_INPUT}}"
}', // 请求体模板
  businessParams: [],                 // 业务参数数组
}
```

**业务参数结构**：

```javascript
{
  key: '',          // 变量标识
  label: '',        // 显示文本
  uiType: 'input',  // 交互类型
  required: false,  // 是否必填
  placeholder: '',  // 占位提示语
  options: [],      // 选项列表（select 类型）
  defaultValue: '', // 默认值
  min: 0,           // 最小值（slider/number 类型）
  max: 100,         // 最大值（slider/number 类型）
  step: 1,          // 步长（slider/number 类型）
}
```

### 4. 出参配置

```javascript
outputVariables: [],    // 输出变量数组
componentBindings: [],  // 组件绑定数组
```

**渲染绑定结构**：

```javascript
{
  componentType: '',    // 组件类型（mainContent/thoughtChain/suggestions）
  variableName: '',     // 变量名称
}
```

### 5. 测试与状态

```javascript
testResponse: null,     // 测试响应
testError: null,        // 测试错误
configStatus: 'incomplete', // 配置状态：incomplete/tested/complete
```

## 四、异常处理

### 1. 输入验证异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|----------|
| 智能体名称为空 | 保存时智能体名称未填写 | 显示错误提示：「智能体名称不能为空」 |
| API 信息不完整 | 保存时 API 名称或地址未填写 | 显示错误提示：「API 名称不能为空」或「API 地址不能为空」 |
| 认证信息缺失 | 选择 API_KEY 或 BEARER_TOKEN 但未填写对应值 | 显示错误提示：「API_KEY 不能为空」或「Bearer Token 不能为空」 |
| 入参配置缺失 | 未配置关联 {{USER_INPUT}} 的入参 | 显示错误提示：「必须配置一个关联了 {{USER_INPUT}} 的入参」 |
| 出参配置缺失 | 未配置「主回复」的出参 | 显示错误提示：「必须配置了「主回复」的出参」 |

### 2. 接口测试异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|----------|
| 接口测试失败 | API 请求失败 | 保存时显示确认提示，说明接口测试失败，允许用户选择继续保存 |
| 接口未测试 | 未进行接口测试 | 保存时显示确认提示，说明接口未测试，允许用户选择继续保存 |

### 3. 系统异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|----------|
| 应用加载失败 | 应用初始化过程中出现错误 | 显示应用加载失败页面，提供「刷新页面」和「重置状态」按钮 |
| 组件渲染错误 | 组件渲染过程中出现错误 | 错误边界捕获错误，显示错误信息，允许用户刷新页面 |

### 4. 网络异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|----------|
| 网络请求失败 | API 测试或保存时网络连接失败 | 显示错误提示，告知用户网络连接失败 |
| 跨域错误 | 访问第三方 API 时出现跨域问题 | 通过 Vercel 代理解决，在 vercel.json 中配置重写规则 |

## 五、业务规则

### 1. 变量替换规则

- **内置变量**：USER_INPUT、SESSION_ID、TIMESTAMP、USER_ID
- **业务变量**：用户定义的业务参数，如 style、max_tokens 等
- **替换时机**：发送 API 请求前，系统扫描模板中的所有 {{变量名}} 并替换为实际值
- **类型处理**：根据值类型进行不同处理（对象、数字/布尔值、字符串）

### 2. 渲染规则

- **主回复内容**：使用 ECHO 万能渲染引擎，支持 Markdown、图片、ECharts 图表
- **思维链**：使用折叠气泡组件，有数据时显示「思考中...」
- **建议问题**：使用快捷标签组件，要求数据为字符串数组

### 3. 配置状态流转

- **incomplete**：初始状态，配置不完整
- **tested**：接口测试通过，配置完整
- **complete**：配置完成并保存

## 六、技术实现

### 1. 前端技术栈

- React 18
- Tailwind CSS
- Lucide React（图标库）
- react-markdown（Markdown 渲染）
- ECharts（图表渲染）

### 2. 状态管理

- 使用 React Context API 进行全局状态管理
- ConfigContext 提供配置数据和操作方法

### 3. 数据持久化

- 目前为模拟保存，实际项目中可使用本地存储或后端 API

### 4. 部署配置

- 使用 Vercel 部署
- 配置 vercel.json 解决跨域问题

## 七、使用场景

### 1. 新增智能体

1. 填写智能体基本信息
2. 配置 API 基本信息
3. 配置入参映射和业务参数
4. 配置出参渲染插槽
5. 测试接口（可选）
6. 保存配置

### 2. 编辑智能体

1. 加载已有配置
2. 修改相应字段
3. 测试接口（可选）
4. 保存配置
